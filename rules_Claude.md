# Rules for Claude — Philipp student PRO Project

## How to use this file
Read this file before taking any action in this project.

---

## Rules

### 1. File naming — all files
Every file Claude creates must end with `_Claude` before the extension, regardless of file type.
**Examples:** `Summary_Claude.md`, `Architecture_Claude.md`, `Rachid_Project_Discussion_Deck_Claude.html`
This allows Jan to distinguish Claude-generated files from his own.

### 2. Read this file first
Before doing anything in this project, read `Rules_Claude.md` to check for relevant instructions.

### 3. Always log changes to the changelog
After every completed task or group of related tasks, add an entry to `changelog_Claude.md`.
- Use today's date as the section header: `## [YYYY-MM-DD] Short description`
- Group changes under `### Added`, `### Changed`, `### Fixed`
- List every file affected
- Never skip this step, even for small changes.

### 4. Always update tasks_Claude.md
After completing any task, mark it `[x]` in `tasks_Claude.md`.
- If a task was added mid-session that wasn't in the list, add it and mark it done.
- Keep the tasks file as an accurate live record — not a historical wishlist.

### 5. Commit all doc updates to GitHub
After updating changelog and tasks, include those files in the git commit alongside the code changes.
Never leave changelog_Claude.md or tasks_Claude.md out of a commit.
